# Glossary

Short explanations for readers. This is not a dictionary; it is a practical guide for how terms are used in this project.

## Agent

An AI system that can decide next steps and use tools, not just answer one message.

## Agentic AI

AI that works in loops: think, choose action, use tool, observe result, continue or stop.

## Tool Calling

When a model asks the system to run a function or tool, such as search, file read, browser action, or API call.

## Context

The information given to the model for the current task: user request, history, files, tool results, rules, and examples.

## Context Engineering

Designing what information the model should see, in what order, and how much, so it can work reliably.

## RAG

Retrieval-Augmented Generation. The system searches external documents or data first, then gives the retrieved information to the model.

## MCP

Model Context Protocol. A way to connect models and tools through a standard server/client pattern.

## Memory

Information saved across turns or sessions so the agent can remember useful facts later.

## Planner

The part of an agent system that breaks a goal into steps.

## Worker Agent

An agent or component that performs a specific task from the plan.

## Critic Agent

An agent or component that reviews output, checks mistakes, and suggests fixes.

## Workflow

A fixed or semi-fixed sequence of steps. It may use AI, but it does not always decide freely like an agent.

## Guardrails

Rules and checks that keep the agent safer, more predictable, and less likely to do harmful actions.

## Coding Agent

An AI assistant that can inspect code, edit files, run commands, and help with software engineering tasks.

## DevOps Automation

Automation around deployment, CI/CD, infrastructure, monitoring, scripts, and operations work.

## Human-in-the-loop

A design where a human reviews or approves important steps before the agent continues.
